## HAMIFlow and HAMINode Implementation Guidelines

### Flow Execution Modes

HAMIFlow classes support two execution modes with different behaviors:

- **Raw Mode**: When `flow.run(shared)` is called directly
- **Wired Mode**: When a flow is used as a node in another flow's orchestration

**Important**: All current HAMIFlow implementations set up their node chaining in the `run()` method. This works correctly in raw mode but **fails silently in wired mode**.

#### Raw Mode Execution
```ts
// Direct execution - works correctly
const flow = new MyFlow(config);
await flow.run(shared);  // run() sets up chaining, then calls _run()
```

#### Wired Mode Execution
```ts
// Used as node in another flow - fails silently
this.startNode.next(myFlow);  // myFlow._run() called directly, bypassing run()
```

### Implementation Requirements

**For HAMIFlow Implementors:**

1. **Move chaining setup** from `run()` to `prep()` method
2. **Ensure shared state initialization** happens in `prep()` or constructor
3. **Test both execution modes** when implementing new flows

**Example Pattern:**
```ts
export class MyFlow extends HAMIFlow<SharedState, Config> {
  async run(shared: SharedState): Promise<string | undefined> {
    // Initialize shared state here
    shared.opts = { verbose: this.config.verbose };
    shared.count = shared.count || 1;

    // Do NOT set up chaining here
    return super.run(shared);
  }

  async prep(shared: SharedState): Promise<void> {
    // Set up node chaining here instead
    const n = shared.registry.createNode.bind(shared.registry);
    this.startNode
      .next(n('some-node'))
      .next(n('another-node'));
  }
}
```
