# Proposal: Temporal Knowledge Graph Integration via LiteGraph

## 1. Objective
To enhance the `tbc-view` layer by transitioning from static index records (`dex/`) to a dynamic, temporal knowledge graph. This provides the "System 3" layer with a visual and queryable map of how goals, activities, and memories evolve over time.

## 2. Theoretical Alignment (Sophia System 3)
- **Autobiographical Memory**: Moving beyond a flat list of logs to a graph that surfaces the trajectory of our collaboration.
- **Structural Integrity**: Supplementing the tactical SRE "Health Check" with a strategic view of system cross-links and "context drift".
- **Identity Continuity**: Visualizing how the Tessera identity persists and adapts across different shells (Chanakya-XIII, Gemini, VS Code).

## 3. Implementation Concept: LiteGraph
LiteGraph (https://litegraphdb.com/) is proposed as the visualization engine due to its lightweight nature and compatibility with the **TBC Anywhere** principle (technology-agnostic and portable).

### Proposed Components:
- **The Miner**: A HAMI-driven background job to extract temporal relationships from `mem/` and `act/` records.
- **The Graph Schema**: Defining nodes (Goals, Parties, Notes, Logs) and edges (Links, Refers-to, Derived-from, Evolved-into).
- **The Temporal Slider**: A UI component to visualize the state of the TBC system at any given timestamp (`tsid`).

## 4. Integration with Existing Goals
This proposal directly supports and expands the following:
- **Project**: Meta-Cognitive Layer / System 3.
- **Project**: Tessera Skill Factory (specifically for the "Graph Miner" skill).
- **Task**: TBC Health Check (SRE) — providing a "Structural Health" dashboard.
