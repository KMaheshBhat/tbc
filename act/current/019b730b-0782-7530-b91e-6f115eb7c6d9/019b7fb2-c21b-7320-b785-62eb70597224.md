### Testing Strategy

Following the TBC Tester Guide (doc/tester-guide.md), all testing must occur in isolated test directories to prevent repository corruption:

- **Test Environment Setup**: Create `_test/` directory structure with fresh TBC root if not preset (do a `tbc sys validate --root ./_test` to verify).
- **Isolation Rule**: Never run CLI commands in repository root; always use `--root ./_test` explicitly.
- **Integration Tests**: Test GraphMinerFlow with new nodes in test root using `tbc dex index --root ./_test`.
- **CLI Command Tests**: Test renamed commands and aliases in test root: `tbc dex index --root ./_test`, `tbc dex health --root ./_test`, etc.
- **Regression Tests**: Ensure existing functionality intact by comparing outputs in test roots.
- **Filesystem Safety**: Verify `git status --porcelain` shows clean working tree after each test.
- **Clean Slate**: Use `rm -rf _test/ && mkdir _test/ && cd _test/ && git init` for reproducible runs.
