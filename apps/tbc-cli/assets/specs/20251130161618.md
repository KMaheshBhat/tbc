# Third Brain Companion System Definition 0.2

The purpose of a Third Brain Companion and its sub-systems is to provide an technology agnostic portable system to conceptualize, operate, and use an Agent for one Prime user or multiple Prime user group.

## Specifications

- this system refers to the Third Brain Companion System
- the system must provide operations specified in [Methods](#methods)
- the system must use plain-text format for operations
- the system may use any implementation for storage
- the system's sub-system of records must be as specified in [Schema](#schema); records may be must be in Markdown format and the front matter will have all fields except content
- the system's sub-system of records must also store the TBC Definition as records
- the fields specified must not be restricted by other systems using or extending this system
- the methods may be restricted by any rules specified in other systems using this system

## Schema

- [vault](./20250507140104.md)
	- the System of Record for the Third Brain Companion
	- the storage will be using `git`
	- the records will have
		- [id](./20250507141631.md)
			- required
			- must be UUID v7 except:
				- the Root record in which case it will be `root`
				- the Definition records 
		- [content](./20250507144249.md)
			- optional
			- generally present
	- there will be a Root Record with following content specification
		- [tbc-root-record](./20251130163650.md)

## Actors

- Agent - the core persona of the Third Brain Companion
- Prime User - the primary user or user group of the Third Brain Companion

## Methods

- All methods specified by the `vault` system specified in [Schema](#schema)
- All methods of `git` can be used by the Prime User
- The Agent may use a method of `git` ONLY AFTER OBTAINING permission from the Prime User
- [instantiate-agent-method](./20251204141919.md)
- [gather-context-method](./20251204141920.md)
- [persist-memories-method](./20251204141921.md)
- [reflection-method](./20251203071809.md)
