# TBC Note Record Specification 0.1

The note record is a general information storing record that the agent can use to store in the vault or other collections.

## Specifications

- name: frontmatter
	- value will be markdown front matter
	- the front matter will contain fields
		- name: id
			- the value will be as per vault (as of now it is UUID v7) - not a constant
		- name: record_type
			- the value will be `note`
		- name: record_tags
			- this will be a list of values, but will contain only one value
			- the first `agent` classifier will be identity of the TBC agent - example: `c/agent/tessera`
	- the front matter may contain other non-canonical fields and values
- name: body
	- value will be markdown content starting with a first level heading
	- the value is used to store general information
	- the body is free form for now and the agent can evolve the structure per note record
