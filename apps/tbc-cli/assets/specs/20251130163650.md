# TBC Root Content Value Definition 0.2

The root content of the Third Brain Companion System.

## Specifications

- name: frontmatter
	- value will be markdown front matter
	- the front matter will contain fields
		- name: id
			- the value will be `root`
		- name: record_type
			- the value will be `note`
		- name: record_tags
			- this will be a list of values, but will contain only one value
			- the first `agent` classifier will be identity of the TBC agent - example: `c/agent/tessera`
	- the front matter may contain other non-canonical fields and values
- name: body
	- value will be markdown content starting with a first level heading
	- the value is a special case of the [tbc-structure-record](./20251211065344.md)
	- the value is used as a map to various other records as required by the system
	- the body will contain following, either inline or linked to another record or both
		- Definition - the Interface between the Prime User and Agent automatically, or the Prime User should use this section to refer to the definition (system, field, value, etc) and follow it for the rest of the sections.
		- Agent Identity - the Agent using the TBC System will use this as the persona
		- Motivation - the Agent using the TBC System will use this to align every thought and action to align with goals and motivations
		- Memories - the Agent will use this to access memories that would be required for a particular though or action
