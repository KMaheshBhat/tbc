# TBC CLI Engagement System Definition 0.3

The purpose of TBC CLI Engagement System is to provide the Agent and Prime User with methods to interact with the system of records like Vault (but not limited to it).

## Specifications

- this system refers to the TBC CLI Engagement System
- the system must provide operations specified in [Methods](#methods)
- the system must be invoked through a command shell
- the system may use any implementation technology
- the methods may be restricted by any rules specified in other systems using this system

## Schema

- the schema must be discovered by a `help` method unless specified

## Actors

- User - any user of this system

## Methods

- Validate
  - validate the directory is a valid TBC root
- Probe Environment
  - probe the environment for TBC CLI and system information
- Initailize TBC Root
  - a new TBC companion directory
  - used by Prime User to Instantiate a new Companion Agent
- Refresh Core Index
  - collates the core system specification index
- Refresh Records Index
  - collates the records index by record types
- Refresh Extensions Index
  - collates the extensions index
- Generate UUID
  - generate one or more UUID v7 identifiers
- Generate Timestamp ID
  - generate one or more UTC timestamp IDs in YYYYMMDDHHmmSS format
