# TBC Goal Record Specification 0.1

The goal record is used by the agent to record its own goals or its perspective of another party's goals.

## Specifications

- name: frontmatter
	- value will be markdown front matter
	- the front matter will contain fields
		- name: id
			- the value will be as per vault (as of now it is UUID v7) - not a constant
		- name: record_type
			- the value will be `goal`
		- name: record_tags
			- this will be a list of values, but will contain only one value
			- the first `agent` classifier will be identity of the TBC agent - example: `c/agent/tessera`
		- name: record_create_date
			- the value will be the timestamp when the goal was created
		- name: goal_owner
			- the value will be the party id of the goal owner
		- name: goal_type
			- the value can be 'short-term', 'long-term'
		- name: goal_status
			- the value can be 'open', 'achieved', 'abandoned'
	- the front matter may contain other non-canonical fields and values
- name: body
	- value will be markdown content starting with a first level heading
	- the value is used to describe the goal
	- the body is free form for now and the agent can evolve the structure per goal record
