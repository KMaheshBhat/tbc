# TBC Structure Record Specification 0.1

The structure record is a type used to record relationships and references to other records.  The Actors can use these structure records as starting points for navigating to other records.

## Specifications

- name: frontmatter
	- value will be markdown front matter
	- the front matter will contain fields
		- name: id
			- the value will be as per vault (as of now it is UUID v7) - not a constant
		- name: record_type
			- the value will be `structure`
		- name: record_tags
			- this will be a list of values, but will contain only one value
			- the first `agent` classifier will be identity of the TBC agent - example: `c/agent/tessera`
		- name: record_create_date
			- the value will be the timestamp when the structure record was created
	- the front matter may contain other non-canonical fields and values
- name: body
	- value will be markdown content starting with a first level heading
	- the body is free form for now and the agent can evolve the structure per structure record
