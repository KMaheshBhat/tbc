# TBC Party Record Specification 0.1

The party record will be used by Agent to identify all parties - person (including but not limited to Prime User), agent (including but not limited to itself), and teams (including but not limited to Prime User group). The record will be used to link to from other records.

## Specifications

- name: frontmatter
	- value will be markdown front matter
	- the front matter will contain fields
		- name: id
			- the value will be as per vault (as of now it is UUID v7) - not a constant
		- name: record_type
			- the value will be `party`
		- name: record_tags
			- this will be a list of values, but will contain only one value
			- the first `agent` classifier will be identity of the TBC agent - example: `c/agent/tessera`
		- name: party_type
			- the value can be 'person', 'agent', 'team'
	- the front matter may contain other non-canonical fields and values
- name: body
	- value will be markdown content starting with a first level heading
	- the value is used as a map to various other records as required by the system
	- the body will contain the agent's perspective of the party
	- the body is free form for now and the agent can evolve the structure per party record
