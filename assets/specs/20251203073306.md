# TBC Log Record Specification 0.1

The log record is a generic type used to record various actions, events, or reflections within the TBC system. For now, it is used to log actions taken during the reflection method.

## Specifications

- name: frontmatter
	- value will be markdown front matter
	- the front matter will contain fields
		- name: id
			- the value will be as per vault (as of now it is UUID v7) - not a constant
		- name: record_type
			- the value will be `log`
		- name: record_tags
			- this will be a list of values, but will contain only one value
			- the first `agent` classifier will be identity of the TBC agent - example: `c/agent/tessera`
		- name: log_type
			- the value will indicate the type of log, e.g., 'reflection', 'interaction'
		- name: log_timestamp
			- the value will be the timestamp when the log was created
	- the front matter may contain other non-canonical fields and values
- name: body
	- value will be markdown content starting with a first level heading
	- the value describes the logged action or event
	- the body is free form for now and the agent can evolve the structure per log record
