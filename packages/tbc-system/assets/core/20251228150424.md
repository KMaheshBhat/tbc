# TBC Operational Fidelity (CLI) Specification 0.4

## Instantiate Companion (`sys init`)

* Allows a Prime User to instantiate a new Companion directory.

```bash
$ tbc sys init --help
Usage: tbc sys init [options]

Initialize a new Third Brain Companion directory

Options:
  --companion <name>  Name of the AI companion
  --prime <name>      Name of the prime user (group)
  -h, --help          display help for command
```

## Upgrade Companion (`sys upgrade`)

* Allows a Prime User to upgrade an existing Companion directory.

```bash
$ tbc sys upgrade --help
Usage: tbc sys upgrade [options]

Upgrade an existing Third Brain Companion directory

Options:
  -h, --help          display help for command
```

## Integrity Validation (`sys validate`)

* Verifies the basic TBC directory structure and key files exist.
* Reports validation status and missing components

## Interface Probing (`int probe`)

* Provides general environment and system information including TBC CLI version, root directory validation, Git repository status, Node.js version, user and host details, system uptime, timestamps, OS information, and shell details.

## Interface Generation (`int`)

* Generates interface configurations for AI assistants to operate as the TBC companion.

```bash
$ tbc int --help
Usage: tbc int [options] [command]

Interface commands

Options:
  -h, --help      display help for command

Commands:
  probe           Probe the environment for TBC CLI and system information
  kilocode        Generate Kilo Code interface configuration
  goose           Generate Goose interface configuration
  github-copilot  Generate GitHub Copilot interface configuration
  help [command]  display help for command
```

## Indexing (`dex`)

* Refreshes non-authoritative summary files in the `dex/` directory based on explicit paths in `root.md`.

```bash
$ tbc dex --help
Usage: tbc dex [options] [command]

Refresh indexes

Options:
  --root <path>   Root directory
  -h, --help      display help for command

Commands:
  core            Refresh the core system definitions index
  records         Refresh all records indexes
  extensions      Refresh extensions index
  help [command]  display help for command
```

## Memory Operations (`mem`)

* Provides direct access to companion and prime user information for quick reference.

```bash
$ tbc mem --help
Usage: tbc mem [options] [command]

Memory operations

Options:
  -h, --help      display help for command

Commands:
  companion       Display companion information
  prime           Display prime user information
  stub            Create stub records for various record types in memory
  help [command]  display help for command
```

### Companion Information (`mem companion`)

* Displays companion ID, name, or full record.

```bash
$ tbc mem companion --help
Usage: tbc mem companion [options]

Display companion information

Options:
  --show <type>   What to show: id (default), name, or full
  -h, --help      display help for command
```

### Prime User Information (`mem prime`)

* Displays prime user ID, name, or full record.

```bash
$ tbc mem prime --help
Usage: tbc mem prime [options]

Display prime user information

Options:
  --show <type>   What to show: id (default), name, or full
  -h, --help      display help for command
```

### Stub Records Creation (`mem stub`)

* Creates a stub record for a specific record type in memory to demonstrate TBC record structures.

```bash
$ tbc mem stub --help
Usage: tbc mem stub [options] <recordType>

Create a stub record for a specific record type in memory

Arguments:
  recordType  Record type to create stub for (party|goal|log|note|structure)

Options:
  -h, --help      display help for command
```
